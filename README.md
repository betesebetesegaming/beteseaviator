# BETESE Aviator

Multi-tenant crash/aviator real-money gaming platform for the West-African market (XOF, mobile-money payments).

**Domain:** beteseaviator.com

## Tech Stack

- **Frontend:** Next.js 16 (App Router) + TypeScript + Tailwind CSS 4 — deployed on **Vercel**
- **Backend:** Firebase
  - **Authentication:** Email/Password, Google, Phone number
  - **Firestore:** users, wallets, transactions, games, game sessions, commissions, payment requests
  - **Realtime Database:** live crash-game round state (multiplier feed)
  - **Cloud Functions:** game engine (bet/cashout settlement), wallet ledger, commission processing, payment webhooks, admin operations
  - **Storage:** user files / KYC documents

## What it is

BETESE Aviator has two completely different kinds of accounts:

1. **Customers (players)** — deposit, bet on crash games, win, withdraw. Sign in with phone number.
2. **Agents (super agents & sub agents)** — marketers, not players. They push sales with a personal referral link (`/register?ref=<username>`) and earn commission on the GGR (profit) generated by the customers they brought in.

BETESE admin manages users, wallets, withdrawal approvals, reports and settings.

## Project layout

```
app/          Next.js App Router pages (customer /play, /agent, /admin)
lib/          Firebase client SDK, auth store, API helpers
functions/    Firebase Cloud Functions (game engine, wallet, commissions, webhooks)
firestore.rules / database.rules.json / storage.rules
```

## Running locally

```bash
npm install
npm run dev          # http://localhost:3000
```

Cloud Functions:

```bash
cd functions
npm install
npm run build
firebase deploy --only functions,firestore:rules,database,storage
```

## Deployment

Pushed to GitHub → Vercel auto-deploys the frontend. Firebase hosts auth, database, storage and functions.
